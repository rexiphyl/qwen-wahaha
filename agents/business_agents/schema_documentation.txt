# Database Schema Documentation
# Business Database: Hotels, Restaurants, and Car Rentals

This document provides a comprehensive description of every table in the business database.
Agents should refer to this file to understand table purposes, relationships, and column meanings.

================================================================================
COMMON TABLES (Shared across all businesses)
================================================================================

## 1. customers
Purpose: Stores information about all customers who use any of the three business services.
Columns:
  - customer_id: Unique identifier for each customer (Primary Key)
  - first_name: Customer's first name
  - last_name: Customer's last name
  - email: Unique email address for communication
  - phone: Contact phone number
  - address: Full street address
  - country: Country of residence
  - registration_date: Date when customer registered
  - loyalty_points: Accumulated loyalty points from all business interactions
Relationships: Referenced by hotel_bookings, restaurant_reservations, restaurant_orders, car_rentals, payments, reviews

## 2. employees
Purpose: Stores information about employees working across hotels, restaurants, and car rental services.
Columns:
  - employee_id: Unique identifier for each employee (Primary Key)
  - first_name: Employee's first name
  - last_name: Employee's last name
  - position: Job title/role
  - hire_date: Date of employment start
  - salary: Annual salary amount
  - department: Business unit (Hotel, Restaurant, Car Rental)
  - contact_number: Employee contact phone
Relationships: Referenced by car_rentals (for the agent who processed the rental)

================================================================================
HOTEL BUSINESS TABLES
================================================================================

## 3. hotels
Purpose: Master table containing information about all hotel properties.
Columns:
  - hotel_id: Unique identifier for each hotel (Primary Key)
  - name: Hotel name
  - address: Street address
  - city: City location
  - country: Country location
  - star_rating: Hotel rating (1-5 stars)
  - description: Marketing description of the property
  - amenities: Comma-separated list of available amenities
  - contact_email: Hotel's contact email
  - contact_phone: Hotel's contact phone
Relationships: Referenced by hotel_rooms

## 4. room_types
Purpose: Defines different categories of rooms available across hotels.
Columns:
  - room_type_id: Unique identifier for room type (Primary Key)
  - type_name: Name of the room type (e.g., "Standard Single", "Deluxe Double")
  - description: Detailed description of the room type
  - base_price_per_night: Standard nightly rate
  - max_occupancy: Maximum number of guests allowed
  - features: Comma-separated list of room features
Relationships: Referenced by hotel_rooms

## 5. hotel_rooms
Purpose: Individual room inventory for all hotels.
Columns:
  - room_id: Unique identifier for each physical room (Primary Key)
  - hotel_id: Foreign key to hotels table
  - room_type_id: Foreign key to room_types table
  - room_number: Room number/identifier within the hotel
  - floor_number: Floor where room is located
  - status: Current status (available, occupied, maintenance)
  - view_type: Type of view from room (City View, Ocean View, etc.)
Relationships: References hotels, room_types; Referenced by hotel_bookings

## 6. hotel_bookings
Purpose: Records all room reservations and stays.
Columns:
  - booking_id: Unique identifier for each booking (Primary Key)
  - customer_id: Foreign key to customers table
  - room_id: Foreign key to hotel_rooms table
  - check_in_date: Scheduled check-in date
  - check_out_date: Scheduled check-out date
  - total_amount: Total cost of the stay
  - booking_status: Current status (confirmed, checked_in, checked_out, cancelled)
  - special_requests: Any special requests from the customer
Relationships: References customers, hotel_rooms; Referenced by hotel_service_usage, payments

## 7. hotel_services
Purpose: Catalog of additional services offered by hotels.
Columns:
  - service_id: Unique identifier for each service (Primary Key)
  - service_name: Name of the service (e.g., "Spa Treatment", "Room Service")
  - description: Service description
  - price: Cost per unit of service
  - availability_hours: Operating hours for the service
Relationships: Referenced by hotel_service_usage

## 8. hotel_service_usage
Purpose: Tracks when guests use additional hotel services.
Columns:
  - usage_id: Unique identifier for each service usage (Primary Key)
  - booking_id: Foreign key to hotel_bookings table
  - service_id: Foreign key to hotel_services table
  - usage_date: Date when service was used
  - quantity: Number of units consumed
  - total_cost: Total charge for this usage
Relationships: References hotel_bookings, hotel_services

================================================================================
RESTAURANT BUSINESS TABLES
================================================================================

## 9. restaurants
Purpose: Master table containing information about all restaurant locations.
Columns:
  - restaurant_id: Unique identifier for each restaurant (Primary Key)
  - name: Restaurant name
  - address: Street address
  - city: City location
  - country: Country location
  - cuisine_type: Type of cuisine served
  - rating: Average customer rating (0-5)
  - opening_hours: Operating hours
  - contact_email: Restaurant contact email
  - contact_phone: Restaurant contact phone
Relationships: Referenced by menu_items, restaurant_reservations, restaurant_orders

## 10. menu_categories
Purpose: Categories for organizing menu items (e.g., Appetizers, Main Courses).
Columns:
  - category_id: Unique identifier for category (Primary Key)
  - category_name: Name of the category
  - description: Category description
Relationships: Referenced by menu_items

## 11. menu_items
Purpose: Individual food and beverage items available at restaurants.
Columns:
  - item_id: Unique identifier for each menu item (Primary Key)
  - restaurant_id: Foreign key to restaurants table
  - category_id: Foreign key to menu_categories table
  - item_name: Name of the dish/drink
  - description: Detailed description of the item
  - price: Selling price
  - is_available: Boolean indicating if item is currently available
  - calories: Calorie count (if known)
Relationships: References restaurants, menu_categories; Referenced by order_details

## 12. restaurant_reservations
Purpose: Records table reservations at restaurants.
Columns:
  - reservation_id: Unique identifier for each reservation (Primary Key)
  - customer_id: Foreign key to customers table
  - restaurant_id: Foreign key to restaurants table
  - reservation_date: Date of reservation
  - reservation_time: Time of reservation
  - party_size: Number of people in the party
  - table_number: Assigned table identifier
  - special_requests: Any special requests from customer
  - status: Current status (confirmed, seated, completed, cancelled)
Relationships: References customers, restaurants; Referenced by restaurant_orders

## 13. restaurant_orders
Purpose: Records all food and beverage orders.
Columns:
  - order_id: Unique identifier for each order (Primary Key)
  - customer_id: Foreign key to customers table
  - reservation_id: Optional foreign key to restaurant_reservations
  - restaurant_id: Foreign key to restaurants table
  - order_date: Timestamp of order placement
  - total_amount: Total order value
  - payment_status: Payment state (pending, paid, refunded)
  - order_type: Type of order (dine_in, takeout, delivery)
Relationships: References customers, restaurant_reservations, restaurants; Referenced by order_details, payments

## 14. order_details
Purpose: Line items for each restaurant order.
Columns:
  - detail_id: Unique identifier for each line item (Primary Key)
  - order_id: Foreign key to restaurant_orders table
  - item_id: Foreign key to menu_items table
  - quantity: Number of items ordered
  - unit_price: Price per unit at time of order
  - total_price: Total price for this line item
  - special_instructions: Any special cooking instructions
Relationships: References restaurant_orders, menu_items

================================================================================
CAR RENTAL BUSINESS TABLES
================================================================================

## 15. car_categories
Purpose: Defines vehicle categories with pricing and specifications.
Columns:
  - category_id: Unique identifier for category (Primary Key)
  - category_name: Category name (Economy, Midsize, Luxury, SUV, etc.)
  - description: Category description
  - daily_rate: Standard daily rental rate
  - seating_capacity: Number of seats
  - transmission_type: Transmission type (manual, automatic)
Relationships: Referenced by cars

## 16. cars
Purpose: Individual vehicle inventory for car rental fleet.
Columns:
  - car_id: Unique identifier for each vehicle (Primary Key)
  - category_id: Foreign key to car_categories table
  - make: Vehicle manufacturer
  - model: Vehicle model
  - year: Manufacturing year
  - license_plate: Unique license plate number
  - color: Vehicle color
  - mileage: Current odometer reading
  - status: Current status (available, rented, maintenance, decommissioned)
  - location: Current location/office of the vehicle
Relationships: References car_categories; Referenced by car_rentals

## 17. car_rentals
Purpose: Records all vehicle rental transactions.
Columns:
  - rental_id: Unique identifier for each rental (Primary Key)
  - customer_id: Foreign key to customers table
  - car_id: Foreign key to cars table
  - employee_id: Foreign key to employees table (agent who processed rental)
  - rental_start_date: Rental start date
  - rental_end_date: Scheduled end date
  - expected_return_date: Expected return date
  - actual_return_date: Actual return date (when vehicle is returned)
  - rental_cost: Base rental cost
  - status: Current status (active, returned, overdue, cancelled)
  - pickup_location: Location where vehicle was picked up
  - return_location: Location where vehicle should be/was returned
Relationships: References customers, cars, employees; Referenced by rental_insurance, rental_additional_services

## 18. insurance_options
Purpose: Available insurance packages for car rentals.
Columns:
  - insurance_id: Unique identifier for insurance option (Primary Key)
  - insurance_name: Name of insurance package
  - description: Coverage description
  - daily_rate: Daily cost of insurance
  - coverage_limit: Maximum coverage amount
Relationships: Referenced by rental_insurance

## 19. rental_insurance
Purpose: Tracks insurance purchases for each rental.
Columns:
  - rental_insurance_id: Unique identifier (Primary Key)
  - rental_id: Foreign key to car_rentals table
  - insurance_id: Foreign key to insurance_options table
  - start_date: Insurance start date
  - end_date: Insurance end date
  - total_cost: Total insurance cost for the rental period
Relationships: References car_rentals, insurance_options

## 20. car_additional_services
Purpose: Extra services available with car rentals (GPS, child seats, etc.).
Columns:
  - service_id: Unique identifier for service (Primary Key)
  - service_name: Name of the service
  - description: Service description
  - price: Daily or one-time cost
Relationships: Referenced by rental_additional_services

## 21. rental_additional_services
Purpose: Tracks additional services purchased with each rental.
Columns:
  - rental_service_id: Unique identifier (Primary Key)
  - rental_id: Foreign key to car_rentals table
  - service_id: Foreign key to car_additional_services table
  - quantity: Number of units
  - total_cost: Total cost for this service
Relationships: References car_rentals, car_additional_services

================================================================================
CROSS-BUSINESS TABLES
================================================================================

## 22. payments
Purpose: Centralized payment records across all three businesses.
Columns:
  - payment_id: Unique identifier for each payment (Primary Key)
  - customer_id: Foreign key to customers table
  - amount: Payment amount
  - payment_method: Method used (credit_card, debit_card, cash, paypal, bank_transfer)
  - payment_date: Timestamp of payment
  - payment_status: Status (pending, completed, failed, refunded)
  - transaction_reference: External transaction ID
  - business_type: Type of business (hotel, restaurant, car_rental)
  - business_id: ID of the specific business entity (hotel_id, restaurant_id, or rental_id)
Relationships: References customers

## 23. reviews
Purpose: Customer reviews and ratings for all businesses.
Columns:
  - review_id: Unique identifier for each review (Primary Key)
  - customer_id: Foreign key to customers table
  - business_type: Type of business being reviewed (hotel, restaurant, car_rental)
  - business_id: ID of the specific business entity
  - rating: Rating score (1-5)
  - review_text: Written review content
  - review_date: Date of review submission
Relationships: References customers

================================================================================
QUERY GUIDELINES FOR AGENTS
================================================================================

Common Query Patterns:

1. HOTEL QUERIES:
   - Join hotel_rooms → hotels → room_types for room details
   - Join hotel_bookings → hotel_rooms → hotels for booking info
   - Join hotel_service_usage → hotel_services for service charges
   - Filter by booking_status for active/completed bookings

2. RESTAURANT QUERIES:
   - Join menu_items → menu_categories → restaurants for full menu
   - Join restaurant_orders → order_details → menu_items for order contents
   - Join restaurant_reservations → restaurants for reservation info
   - Filter by order_type for dine_in/takeout/delivery analysis

3. CAR RENTAL QUERIES:
   - Join cars → car_categories for vehicle details
   - Join car_rentals → cars → car_categories for rental info
   - Join rental_insurance → insurance_options for insurance details
   - Join rental_additional_services → car_additional_services for extras
   - Filter by status for active/returned/overdue rentals

4. CROSS-BUSINESS QUERIES:
   - Use payments table with business_type filter for revenue analysis
   - Use reviews table with business_type filter for ratings analysis
   - Join customers with any business table for customer history

Key Metrics Tables:
   - Revenue: payments (all businesses), hotel_bookings (hotel), restaurant_orders (restaurant), car_rentals (car rental)
   - Utilization: hotel_rooms.status, cars.status
   - Customer Activity: customers joined with any booking/order/rental table
